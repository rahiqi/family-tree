# Agent Instructions: Project Treely (Full-Scale MongoDB SaaS with EF Core)

You are an expert AI software engineer tasked with building **Treely**, a fluid, multi-lingual family tree SaaS application. The architecture isolates the frontend and backend workspace into dedicated folders using MongoDB via EF Core as its absolute data store.

---

## 1. Project Directory Schema & Stack
* **Repository Root:** Contains `docker-compose.yml` (orchestrating the app container and a local MongoDB instance) and this `Agents.md` file.
* **Backend (`/back`):** .NET 10 Core Web API using the standard Visual Studio project template. It utilizes the official Entity Framework Core provider for MongoDB (`Microsoft.EntityFrameworkCore.MongoDB`).
* **Frontend (`/front`):** React.js initialized via Vite. 
* **Primary UI Tree Library:** `family-chart` (v2) by Donatso (`https://github.com/donatso/family-chart`).
* **Localization:** Multi-language support using `i18next` / `react-i18next`. **Default language is Persian (fa)** with complete Right-to-Left (RTL) layout support.

---

## 2. Technical Context & Reference: `family-chart` (v2)
* **Canvas Operation:** The `family-chart` library natively manages relationship linkages (`parents`, `spouses`, `children`) and node structural integrity on the client side. The React application periodically bulk-saves this complete pristine JSON array block straight into the tree document.
* **Canvas Guardrails:** Instantiate the canvas layout instance inside a standard `useEffect` block hook. Clean up DOM side-effects and explicit SVG containers properly on unmount to prevent double-rendering artifacts caused by React StrictMode.
* **Redirection Linkage:** Modify default node card templates to route click actions or inject anchors pointing directly to: `/tree/:treeId/profile/:id`.

5. Security & Role-Based Access Control (RBAC)
Secure all backend API controller endpoints using standard JWT Bearer Authorization tokens. When processing requests aimed at /api/tree/{treeId}/*, validate the user context against these permission rules:

visitor: Granted execution rights only for HTTP GET endpoints. Frontend canvas workspace must lock or throw read-only flags hiding editing buttons.

editor: Granted permissions to modify the graph arrays and individual profiles via PUT / POST / DELETE methods.

owner: Retains absolute editor privileges plus the unique authorization to add new users or modify relationship collaboration records inside the Collaborators list.

6. Coding Instructions for the Agent
Docker Integration First: Ensure docker-compose.yml sets up MongoDB correctly and configures the connection string properly inside /back appsettings.json.

Pass-Through Stream Layout: Bypass heavy complex entity change-tracking for internal graph links. Accept the frontend family-chart layout payload and save it directly as a serialized text block or primitive document array into MongoDB.

RTL System Architecture: When configuring styling rules via Tailwind or component CSS layouts, do not employ absolute left/right adjustments.
